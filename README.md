# Retroalimentación docente — Proyecto PM I–VI + PM6_CalDif

Repositorio de trabajo colegiado para registrar, clasificar y dar seguimiento a observaciones docentes sobre la propuesta de libros de **Pensamiento Matemático I–VI** y la ruta **PM6_CalDif** de Preparatorias La Salle Bajío.

## Propósito

Recoger observaciones pedagógicas, didácticas, curriculares, lasallistas y normativas sobre la propuesta guía, con el fin de identificar fortalezas, áreas de mejora, ajustes necesarios y recomendaciones concretas para el desarrollo editorial, pedagógico y curricular.

## Estructura sugerida de trabajo

Este repositorio usa **GitHub Issues** como sistema de captura y seguimiento. Cada observación se registra como un issue independiente. Esto permite:

- clasificar observaciones por asignatura;
- etiquetar prioridad;
- discutir ajustes;
- asignar responsables;
- cerrar observaciones cuando se integren;
- conservar trazabilidad de acuerdos.

## Plantillas disponibles

En la pestaña **Issues > New issue** aparecen las siguientes plantillas:

1. **Observación puntual**
   - Para registrar una observación específica sobre una unidad, tema, actividad, evaluación o sección del documento.

2. **Revisión general por asignatura**
   - Para entregar una revisión más amplia de PM I, PM II, PM III, PM IV, PM V, PM VI o PM6_CalDif.

3. **Acuerdo de reunión**
   - Para documentar acuerdos, responsables, fechas y evidencias esperadas.

## Flujo recomendado

1. Crear un issue por cada observación relevante.
2. Seleccionar asignatura, eje y prioridad.
3. Justificar la observación.
4. Proponer un ajuste concreto.
5. Etiquetar o revisar la prioridad.
6. Discutir en comentarios si se requiere.
7. Cerrar el issue cuando el ajuste haya sido integrado o descartado con justificación.

## Uso con GitHub Pages

La carpeta `docs/` contiene una página sencilla de bienvenida. Puede activarse desde:

**Settings > Pages > Build and deployment > Source: Deploy from a branch > Branch: main > Folder: /docs**

La página pública servirá como puerta de entrada para docentes y revisores.

## Nota operativa

GitHub Issues requiere que los participantes tengan cuenta de GitHub. Si se desea evitar esa barrera, se recomienda que un relator capture las observaciones de docentes durante la reunión y las registre posteriormente como issues.
